  Representational State Transfer，簡稱REST，它並不是一種標準而是一種設計風格，原本的API為:
   獲得資料   /getData
   新增資料   /createData
   刪除資料   /deleteData/1
  由工程師自定義，沒有統一的用法，每當要使用不同公司的API都要去document確認，理解命名設計規則才可以使用，然而RESTful API 為:
   獲得資料GET     /data
   新增資料POST    /data
   刪除資料DELETE  /data/1
  就是用一個唯一的 URL 定位資源，將動作藏在 HTTP 的 method 裡面，有以下優點:
